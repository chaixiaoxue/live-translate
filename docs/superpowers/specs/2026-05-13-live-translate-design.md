# Live Translate - 实时会议翻译工具设计文档

## 概述

一个 Windows 本地桌面程序，能够捕获系统音频（如 Teams 会议中的英文语音），实时识别为英文文本，翻译为中文，通过悬浮窗显示翻译结果。

**目标用户**：英文听力有限，需要在英文会议中获得实时中文翻译辅助的用户。

**核心价值**：全本地离线运行，无需联网，保护会议隐私。

## 需求

| 需求项 | 决定 |
|--------|------|
| 运行平台 | Windows |
| 实时性 | 逐句实时，延迟 1-3 秒 |
| 处理方式 | 全本地离线（无云端依赖） |
| 界面形式 | 桌面悬浮窗，始终置顶 |
| 硬件要求 | 无 NVIDIA GPU，纯 CPU 运行 |

┌────────┴────────────────────────────────────────┐
│               翻译处理管线 (Pipeline)              │
│                                                  │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐    │
│  │ 音频捕获  │──▶│ 语音识别  │──▶│  文本翻译 │    │

**数据流**：系统音频 → 环形缓冲区 → VAD 语音端点检测切句 → Whisper 识别英文 → 翻译为中文 → 推送到悬浮窗

## 模块设计

### 1. 音频捕获模块 (audio/capture.py, audio/buffer.py, audio/vad.py)

**技术选型**：WASAPI Loopback（Windows 内置 API，无需安装虚拟音频驱动）

**依赖库**：`pyaudiowpatch`（Windows 专用的 PyAudio 分支，支持 WASAPI Loopback）

**音频参数**：
- 源格式：系统音频输出（通常 48kHz 立体声）
- 目标格式：16kHz 单声道（Whisper 要求）
- 重采样：使用 `numpy` 进行下采样和声道混合

**环形缓冲区**：
- 容量：30 秒音频（防止内存溢出）
- 写入：音频捕获线程持续写入
- 读取：VAD 检测到完整语音段后读取

**VAD 语音端点检测**：
- 使用 `webrtcvad` 库（Google 的 VAD 算法，轻量且 CPU 友好）
- 检测参数：
  - 帧大小：30ms
  - VAD 灵敏度：2（中等，0-3 可调）
  - 静音阈值：800ms（说话人停顿超过 800ms 视为一句结束）
- 输出：切分出的完整语音段（1-15 秒），送入 STT 模块

**处理流程**：
```
耳机输出音频 (48kHz 立体声)
    ↓ 重采样
16kHz 单声道
    ↓ 写入环形缓冲区
VAD 检测语音段
    ↓ 检测到静音 (>800ms)
切出一段完整语音
    ↓ 送入 STT
```

### 2. 语音识别模块 (stt/whisper_engine.py)

**技术选型**：`faster-whisper`（Whisper 的 CTranslate2 优化版本，CPU 性能最佳）

**模型选择**：

| 模型 | 大小 | CPU 速度 | 准确度 | 适用场景 |
|------|------|----------|--------|----------|
| `tiny` | 75MB | ~0.3x RT | 一般 | 低端机备选 |
| **`base`** | **140MB** | **~0.5x RT** | **良好** | **日常推荐** |
| `small` | 460MB | ~1.5x RT | 很好 | 高准确度场景 |

> RT = Real Time。0.5x RT 表示 10 秒音频需要 5 秒处理。

**默认使用 `base` 模型**：体积小（140MB），CPU 上处理速度快，会议口语识别准确度足够。

**关键设计**：
- **模型预加载**：程序启动时将模型加载到内存（~300MB RAM），避免首次识别延迟
- **语言锁定**：`language="en"`，跳过语言检测，节省时间
- **逐句识别**：每段 VAD 切出的音频独立识别，不累积
- **输出**：英文原始文本，同时送入翻译模块和 UI 显示

### 3. 翻译模块 (translate/argos_engine.py)

**技术选型**：`argostranslate`（纯本地离线翻译引擎）

**模型**：en → zh 翻译包，约 100MB

**关键设计**：
- **首次运行自动下载**：检测到无翻译包时自动下载 en→zh 包，显示进度条
- **翻译速度**：CPU 上 <0.5 秒/句，不会成为瓶颈
- **逐句翻译**：与 STT 输出对齐，每句英文独立翻译
- **保留原文**：界面同时显示英文原文和中文翻译，方便对照

**备选方案**：`Helsinki-NLP/opus-mt-en-zh`（Hugging Face），翻译质量稍好但需手动转换格式。

### 4. 悬浮窗 UI (ui/floating_window.py, ui/settings_dialog.py, ui/tray_icon.py)

**技术选型**：PyQt5

**界面布局**：
```
┌─────────────────────────────────────────────┐
│  ⏺ Live Translate         ─ □ ×            │  ← 标题栏（可拖拽）
├─────────────────────────────────────────────┤
│                                             │
│  🎤 Hello everyone, let's start the        │  ← 最新英文原文
│     meeting and discuss Q2 results.         │
│                                             │
│  📝 大家好，我们开始开会，讨论一下           │  ← 中文翻译
│     第二季度的业绩。                         │
│                                             │
│  ─────────────────────────────────────────  │  ← 分隔线
│  🎤 I think we should focus on the new     │  ← 上一句英文
│     product launch timeline.                │
│  📝 我认为我们应该关注新产品                 │  ← 上一句翻译
│     发布的时间线。                           │
│                                             │
├─────────────────────────────────────────────┤
│  [▶ 开始]  [⏸ 暂停]  [⚙ 设置]       85%   │  ← 控制栏（85% 为 CPU 使用率）
└─────────────────────────────────────────────┘
```

**核心功能**：
- **始终置顶**：`Qt.WindowStaysOnTopHint`，不会被 Teams 窗口遮挡
- **半透明背景**：可调透明度（30%-90%），不完全遮挡后面内容
- **可拖拽**：鼠标拖动标题栏移动位置
- **可调大小**：拖拽边框调整窗口尺寸
- **自动滚动**：默认显示最近 3 条翻译结果（可在设置中调整为 1-10 条），新结果自动替换旧结果
- **开始/暂停**：一键暂停翻译
- **系统托盘**：关闭窗口时最小化到托盘，不退出程序

**设置面板**：
- Whisper 模型选择（tiny / base / small）
- 透明度滑块（30%-90%）
- 字体大小调节
- 显示条数（1-10 条，默认 3 条）
- 窗口位置和大小记忆（下次启动恢复）

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 系统音频无输出 | 界面提示"等待音频..."，持续监听 |
| Whisper 模型加载失败 | 提示用户检查磁盘空间，回退到 tiny 模型 |
| 翻译包未下载 | 首次启动自动下载，显示进度条 |
| CPU 占用过高 | 自动降低 Whisper 模型大小，或增加 VAD 静音阈值 |
| 音频设备切换 | 自动检测默认音频设备变化，重新连接 |

## 线程设计

程序采用多线程架构，避免 UI 卡顿：

```
主线程 ─── PyQt5 UI 事件循环
  │
  ├── 音频捕获线程 ─── 持续捕获系统音频，写入环形缓冲区
  │
  └── 处理线程 ─── VAD 切句 → Whisper 识别 → Argos 翻译 → 信号推送 UI
```

- **音频捕获线程**：独立线程持续捕获音频，写入环形缓冲区
- **处理线程**：从缓冲区读取音频，执行 VAD → STT → 翻译管线，通过 Qt 信号将结果推送到 UI
- **主线程**：仅负责 UI 渲染和用户交互

## 打包分发

使用 `PyInstaller` 打包为 Windows 可执行文件：
- 单文件 `.exe` 或目录分发
- 预打包 Whisper base 模型和 Argos en→zh 翻译包
- 预期体积：~500MB（含模型）
- 首次启动无需下载任何内容
